# OSA Prompt Examples

This document shows example prompts generated by `buildWelcomeAppPrompt()` for different business types and onboarding scenarios.

## Example 1: Agency (Client Management Focus)

**Onboarding Data:**
- Workspace Name: "Acme Agency"
- Business Type: "agency"
- Team Size: "2-5"
- Role: "founder"
- Challenge: "managing multiple client projects simultaneously"
- Integrations: ["hubspot", "slack", "notion"]

**Generated Prompt:**
```
Create a welcome workspace application for Acme Agency.

Business Context:
- Business Type: agency
- Team Size: 2-5
- User Role: founder
- Main Challenge: managing multiple client projects simultaneously

Requirements:
- Create a simple, user-friendly dashboard
- Include getting started guide and quick actions
- Add client management features
- Include project tracking capabilities

Integrations to highlight: hubspot, slack, notion

Technical Stack:
- Backend: Go with PostgreSQL
- Frontend: Modern web UI with responsive design
- Follow BusinessOS architecture patterns
```

## Prompt Customization Logic

### Business Type → Feature Mapping

| Business Type | Features Added |
|---------------|----------------|
| **agency** | Client management + Project tracking |
| **consulting** | Client management + Project tracking |
| **startup** | Product roadmap + Team collaboration |
| **freelance** | Time tracking + Invoice/payment tracking |
| **other** (default) | Task management + Team overview |

### Integration Highlighting

If integrations are selected, they are added to the prompt:
```
Integrations to highlight: slack, linear, notion
```

This helps OSA understand which third-party tools the user wants to connect.

### Technical Stack (Always Included)

Every prompt includes BusinessOS technical requirements:
- Backend: Go with PostgreSQL
- Frontend: Modern web UI with responsive design
- Follow BusinessOS architecture patterns

This ensures generated apps are compatible with the BusinessOS ecosystem.

## Testing Prompts

To test prompt generation with different scenarios:

1. Complete onboarding with different business types
2. Check server logs for generated prompts:
   ```bash
   grep "buildWelcomeAppPrompt" server.log
   ```
3. Verify customization matches business type
4. Test with minimal data (skip flow)
5. Test with all fields populated

## Future Enhancements

### Planned Improvements:

1. **Template Variations**
   - Industry-specific templates (healthcare, finance, education)
   - Team size-specific templates (solo, small team, enterprise)

2. **Challenge-Based Customization**
   - Parse challenge text for keywords
   - Add challenge-specific features
   - Example: "sales" → Add CRM features

3. **Integration-Aware Prompts**
   - If HubSpot selected → Add CRM features
   - If Linear selected → Add project management
   - If Slack selected → Add team chat integration

4. **User Role Customization**
   - Developer → Code-focused features
   - Manager → Reporting dashboards
   - Sales → Pipeline management

5. **Multi-Language Support**
   - Detect user language from onboarding
   - Generate prompts in user's language
