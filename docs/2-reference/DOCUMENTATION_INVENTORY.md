```bash
# Search for keyword in all markdown files
grep -r "keyword" docs/ --include="*.md"

# Find all markdown files
find docs/ -name "*.md" -type f

# Find files modified in last 7 days
find docs/ -name "*.md" -mtime -7
```

### Method 4: Use Git
```bash

git log --oneline -- docs/ | head -20

# See what changed in a specific doc
git log -p -- docs/filename.md
```

## 📝 Standards for Documentation

### File Naming
- Use UPPERCASE_WITH_UNDERSCORES for root level
- Use lowercase-with-dashes for subdirectories
- Include dates for reports: 2026-01-19-report-name.md
- Use descriptive names that indicate content

### Location
- New feature docs → `/docs/1-active/features/feature-name/`
- Development guides → `/docs/development/category/`
- Architecture → `/docs/architecture/`
- Reports → `/docs/1-active/reports/` or `/docs/reports/`
- Archive old docs → `/docs/archive/`

### Metadata
- Include "Last Updated" date at top
- Include "Maintained By" for active docs
- Include "Purpose" or "Summary" section
- Include table of contents for long docs

### Linking
- Use relative paths: `../other-doc.md`
- Link to entry point: `../0-START-HERE/README.md`
- Link to index: `../INDEX.md`
