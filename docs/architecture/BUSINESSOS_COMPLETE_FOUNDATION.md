### 1.3 The Core Philosophy

> **"Make users feel like they never left home"**

This means:
1. **Familiar interfaces** - Adopt UI/UX patterns that won the market
2. **Same workflows** - Don't reinvent, integrate and enhance
3. **Zero learning curve** - If they used Notion, they know our Pages
4. **Everything connects** - But smarter, with AI understanding context

### 4.3 Module Data Models

#### Email Module (Needed)
```sql
emails
├── id, user_id
├── integration_id (gmail, outlook)
├── thread_id
├── message_id (external)
├── from_address, to_addresses, cc_addresses, bcc_addresses
├── subject, body_text, body_html
├── attachments (JSONB)
├── labels (JSONB array)
├── is_read, is_starred, is_draft, is_sent
├── sent_at, received_at
├── client_id → linked client
├── project_id → linked project
└── ai_summary, ai_priority

email_threads
├── id, user_id
├── subject
├── participants (JSONB)
├── message_count
├── last_message_at
└── client_id, project_id
```

#### Files Module (Needed)
```sql
files
├── id, user_id
├── name, path
├── mime_type, size_bytes
├── storage_provider (local, google_drive, onedrive, s3)
├── external_id
├── parent_folder_id → hierarchy
├── is_folder
├── shared_with (JSONB)
├── context_id → linked page
├── project_id → linked project
├── client_id → linked client
└── ai_summary, ai_tags

file_versions
├── id, file_id
├── version_number
├── size_bytes
├── uploaded_by
└── created_at
```

#### Automations Module (Needed)
```sql
automations
├── id, user_id
├── name, description
├── trigger_type (webhook, schedule, event, manual)
├── trigger_config (JSONB)
├── steps (JSONB array)
├── is_active
├── last_run_at
├── run_count
└── error_count

automation_runs
├── id, automation_id
├── status (running, success, failed)
├── started_at, completed_at
├── input_data (JSONB)
├── output_data (JSONB)
├── error_message
└── step_results (JSONB)
```

#### Forms Module (Needed)
```sql
forms
├── id, user_id
├── name, description
├── fields (JSONB array)
├── settings (JSONB)
├── thank_you_message
├── redirect_url
├── is_published
├── response_count
├── project_id → linked project
└── context_id → linked page

form_responses
├── id, form_id
├── data (JSONB)
├── submitted_at
├── ip_address
├── user_agent
└── client_id → linked client (if identified)
```

#### Meetings Module (Needed)
```sql
meetings
├── id, user_id
├── calendar_event_id
├── title
├── participants (JSONB)
├── recording_url
├── transcript
├── summary (AI-generated)
├── action_items (JSONB)
├── highlights (JSONB array)
├── duration_seconds
├── started_at, ended_at
├── context_id → meeting notes page
├── project_id → linked project
└── client_id → linked client
```

#### Notifications Module (Needed)
```sql
notifications
├── id, user_id
├── type (mention, assignment, reminder, system)
├── title, message
├── source_type (task, project, page, etc.)
├── source_id
├── actor_id (who triggered it)
├── is_read
├── read_at
├── is_dismissed
└── created_at
```

#### Search Index (Needed)
```sql
search_index
├── id
├── entity_type (page, task, project, client, etc.)
├── entity_id
├── user_id
├── title
├── content_text (searchable)
├── tags (JSONB array)
├── metadata (JSONB)
├── created_at
├── updated_at
└── search_vector (tsvector for full-text search)
```

## Appendix B: Integration API Reference

### OAuth Flow (Standard)

```
1. User clicks "Connect [Integration]"
2. Redirect to provider OAuth page
3. User authorizes
4. Callback to /api/integrations/[provider]/callback
5. Store tokens in credential_vault
6. Create user_integration record
7. Initial sync
```

### Data Sync Pattern

```go
// Every integration follows this pattern
type IntegrationHandler interface {
    // OAuth
    GetAuthURL(state string) string
    HandleCallback(code string) (*Credentials, error)
    RefreshToken(creds *Credentials) (*Credentials, error)

    // Data operations
    Sync(userID string, options SyncOptions) error
    Push(userID string, data interface{}) error
    Pull(userID string, since time.Time) ([]Record, error)

    // Webhook handling
    HandleWebhook(payload []byte) error
}
```
