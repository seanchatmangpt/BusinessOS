================================================================================
                       USAGE EXAMPLES
================================================================================

1. Initialize Healthcare Workspace:
   $ bos healthcare init --organization "Stanford Medical"
   
   Returns:
   {
     "workspace": "healthcare-stanford medical",
     "ontology_version": "1.0.0-phi",
     "fhir_compatible": true,
     "compliance_frameworks": ["HIPAA", "GDPR", "HITECH"]
   }

2. Track Patient PHI Lineage:
   $ bos healthcare phi track \
       --patient-id pat-001 \
       --phi-elements name,ssn,diagnosis,medication \
       --storage-location postgres
   
   Returns:
   {
     "tracking_id": "PHI-pat-001-1711353000",
     "patient_id": "pat-001",
     "phi_elements": 4,
     "lineage_depth": 3,
     "audit_entries": 5,
     "status": "tracking PHI in postgres"
   }

3. Check Patient Consent (ODRL):
   $ bos healthcare consent check \
       --patient-id pat-001 \
       --resource Observation \
       --access-type read
   
   Returns:
   {
     "patient_id": "pat-001",
     "resource_type": "Observation",
     "consent_valid": true,
     "constraints": ["purpose_limited", "anonymization"],
     "expiration": "2026-12-31",
     "enforcement_status": "ODRL policy active"
   }

4. Verify HIPAA Compliance:
   $ bos healthcare hipaa verify \
       --organization "Stanford Medical" \
       --audit-depth 5000
   
   Returns:
   {
     "organization": "Stanford Medical",
     "verification_date": "2026-03-25T15:30:00Z",
     "compliant": true,
     "findings": [
       "All access logs timestamped with microsecond precision",
       "User identification and authentication verified",
       "Access request content captured",
       "Access response documented",
       "Modification tracking enabled"
     ],
     "retention_years": 6,
     "audit_trail_entries": 5000,
     "issues": []
   }
