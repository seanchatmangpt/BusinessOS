SECURITY FEATURES VERIFIED:
===========================

1. JWT AUTH MIDDLEWARE (HS256)
   ✅ Validates Authorization: Bearer <token> header
   ✅ Verifies signature using secret key
   ✅ Rejects expired tokens
   ✅ Rejects invalid format
   ✅ Rejects wrong algorithm
   ✅ Stores claims in context
   ✅ Returns 401 on failure with specific error codes

2. SESSION AUTH MIDDLEWARE (Cookies)
   ✅ Validates better-auth.session_token cookie
   ✅ Queries Better Auth tables in database
   ✅ Sliding window refresh (extends if < 24h remaining)
   ✅ Absolute max age (30 days, forces re-auth)
   ✅ Stores user in context
   ✅ Returns 401 on failure

3. REQUIRE AUTH MIDDLEWARE (Guard)
   ✅ Enforces that user/claims exist in context
   ✅ Returns 401 if missing
   ✅ Allows request to proceed if authenticated

4. ERROR HANDLING
   ✅ 401 Unauthorized for all auth failures
   ✅ Specific error codes:
     - JWT_MISSING: No Authorization header
     - JWT_INVALID: Invalid/expired token
     - JWT_INVALID_FORMAT: Wrong Bearer format
     - UNAUTHENTICATED: No user in context
   ✅ No information leakage in errors

5. RATE LIMITING
   ✅ 10 req/min on auth endpoints
   ✅ 10 req/min on credential endpoints
   ✅ Prevents brute-force attacks
   ✅ Prevents callback spam

6. CSRF PROTECTION
   ✅ Token endpoint: /api/auth/csrf
   ✅ Validates on state-changing requests
   ✅ Configured skip list for webhooks/health
   ✅ Dev/prod mode handling
